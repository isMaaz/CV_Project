def normalise_slice(slc):
    lo, hi = percentile values at 1 and 99
    return clipped image scaled to 0..1

def preprocess_slice(slc):
    slc = normalise_slice(slc)
    slc = gaussian(slc, sigma=0.65)
    slc = median_filter(slc, size=3)
    slc = uniform_filter(slc, size=2)
    return clipped image

Primary path

def context_from_volume(vol, z):

        take neighboring slice z + dz

    return stack of 5 slices

A single slice has shape 96 x 96 x 1.

3D context input

A tumor region is not isolated to one slice. Neighboring slices provide continuity.

Tensor

After Conv3D, feature maps are reshaped into small feature tokens. Transformer attention compares these tokens.

Dense Layer

Sigmoid Output

26. Common Examiner Follow-Up Questions

Why not YOLO?

Safe answer: YOLO is mainly for bounding-box localization. Our available final labels were segmentation masks, so we used tumor presence recognition with masks and a 3D transformer.

Why use slices instead of full volume?

Safe answer: Full 3D volumes are memory-heavy in Colab. We used 5-slice context as a practical middle path.

Why use binary crossentropy?

Safe answer: The final recognition task has two classes, tumor and no_tumor, so binary crossentropy is appropriate for sigmoid output.

Why use Adam?

Safe answer: Adam is a common optimizer that adapts learning rates for weights and works well for deep models with limited tuning.

Why use dropout?

Safe answer: Dropout reduces overfitting by preventing the model from relying too much on specific neurons.

Why validation accuracy low?

Safe answer: The validation set was small and difficult. It shows generalization risk, so we should not overclaim clinical reliability.

Why use fallback mask if masks exist?

Safe answer: It makes the notebook robust. In this final run masks existed, so labels came from segmentation masks.

Why tumor recall is 1.0?

Safe answer: In the test set, the model found all tumor samples. But it also produced false positives, so precision was lower.

Why PR AUC is modest?

Safe answer: The test set is small and probability ranking is not perfect. The model caught tumor cases but confidence calibration can improve.

How to improve?

Safe answer: Use more subjects, stronger augmentation, cross-validation, better architecture tuning, and external test data.
