Build a high-level application using deep neural networks for object detection and semantic understanding.

Our implementation

We used paired segmentation masks to identify tumor presence, exported image-mask-overlay samples, extracted descriptors, and trained a 3D transformer for recognition.

Main activations

ReLU in Conv3D and Dense layer, GELU inside transformer feed-forward blocks, and sigmoid at output.

Layer count

The network has 13 main stages if transformer encoder blocks are counted as blocks. Inside each transformer block, there is multi-head attention, layer normalization, and two Dense layers.

Neurons and filters

Conv3D filters are 16 then 32. Transformer feed-forward units are 64 then 32 in each block. Final Dense layer has 96 neurons and output has 1 neuron.

Trainable parameters

The model has roughly 35.5k trainable parameters based on these layer settings.

Training, Validation, and Fit Diagnosis

90.65%

8

Batch size

Batch size: The final 3D transformer was trained with batch size 8. This means the model processed 8 samples at a time before updating its weights. A small batch size was used because each sample contains 5 MRI slices, so memory usage is higher than a normal single-image input.
Fit typePatternMeaning
UnderfittingTrain low, validation lowModel has not learned enough.
Good fitTrain and validation closeModel generalizes reasonably.
OverfittingTrain high, validation much lowerModel may memorize training patterns.
Our diagnosis: The model is not underfitting. It shows possible overfitting or generalization risk because training accuracy is much higher than validation accuracy.

Safe claims

Limitations and Future Improvements

Small data

158 samples are enough for academic demonstration, but not enough for clinical deployment.

Generalization risk

Training accuracy was high while validation accuracy was low, showing possible overfitting.

False positives

7 no_tumor samples were predicted as tumor, lowering precision.

Medical safety

This is a semester project prototype, not a diagnostic system.

Future work

  • Use more subjects and more MRI modalities.
  • Apply cross-validation and external test data.
  • Tune model size, dropout, learning rate, and augmentation.
  • Extend from tumor presence recognition to full tumor subregion segmentation.
Conclusion

Final Summary

The final project successfully integrates acquisition, pre-processing, segmentation, description, and recognition into one runnable deep vision workflow.

Data

158 annotated MRI slice samples exported with image, mask, overlay, label, and features.

Model

3D transformer using five neighboring slices and attention over voxel-patch tokens.

Result

78.13 percent test accuracy with 100 percent tumor recall on the held-out test set.

Safe claim

Promising academic prototype with clear need for more data and stronger validation.

Closing line: This pipeline demonstrates how classical computer vision stages and deep learning can be combined for semantic understanding of medical MRI slices.
